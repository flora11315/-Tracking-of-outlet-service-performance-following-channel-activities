# 渠道活动后网点服务效果追踪 Agent 思路记录

## 一、课题方向

本课题拟围绕“渠道活动后网点服务效果追踪”展开。这里的“客户”不是个人投资者，而是渠道客户，即各分行、支行和网点。

课题重点不是将客户服务与销量直接挂钩，也不是评价单个销售或单个终端客户，而是在合规边界内，基于渠道聚合数据观察活动后 T+7、T+30、T+90 三个节点的活动反馈、服务承接、保有量变化、网点客群盈利体验和渠道反馈，并自动提示渠道销售如何优化后续投教陪伴与渠道服务策略。

建议定位为：

**渠道活动后网点服务效果追踪 Agent**

也可以作为对外展示名称：

**渠道活动客户体验追踪 Agent**

核心目标是把渠道投教活动从一次性触达，转化为“活动后及时承接、短期经营观察、中期复盘沉淀”的渠道服务闭环。

## 二、合规与隐私边界

公司合规管理严格，因此方案全流程需要强调以下边界：

1. **客户服务与销量不直接挂钩**
   - 保有量、申购、赎回等数据可以作为经营现象和服务效果的观察指标。
   - 这些指标不作为评价客户服务质量的唯一依据。
   - 输出建议应指向投教陪伴、风险提示、内容优化和渠道支持，而不是直接推动销售。

2. **不使用个人客户 ID**
   - 本课题以渠道客户为分析对象，即分行、支行、网点等。
   - 不设计个人客户 ID、姓名、手机号、身份证号、真实账号等字段。
   - 不追踪单个终端投资者，也不输出针对个人客户的提醒。

3. **使用渠道聚合口径**
   - 输入数据尽量以分行、支行、网点、活动场次、活动类型等维度汇总。
   - 如无法取得某些明细数据，系统应自动降级为更粗颗粒度分析。
   - 输出结果面向渠道销售和渠道服务，不面向个人投资者画像。

4. **表达口径服务化**
   - 不使用“促成申购”“推动转化”等强销售导向措辞作为核心目标。
   - 推荐使用“加强投教陪伴”“补充市场观点材料”“开展风险收益特征解释”“优化活动后跟进机制”等表述。

## 三、业务背景与痛点

常见渠道活动流程通常包括：

- 确定场地，例如会议室、银行网点、分行会场等。
- 了解渠道想听的内容。
- 邀请讲师，包括银行、基金公司、投研人员、指数团队等。
- 举办客户活动，例如投教讲座、信用卡主题活动、银行自有活动等。
- 通过插花、权益活动、专题讲座等方式吸引客户参与。

现有活动评估容易停留在较短期、较表层的指标上，例如：

- 活动办了多少场。
- 到场人数有多少。
- 现场反馈好不好。
- 问卷评分高不高。

但这些指标不能充分回答更重要的问题：

- 活动后网点客户服务效果是否改善？
- 活动后不同阶段的服务承接和渠道反馈是否被及时捕捉？
- 活动后 30 天和 90 天相关网点保有量是否稳定或提升？
- 网点客群整体盈利体验是否改善？
- 哪类银行、哪类网点、哪类活动规模和主题更适合持续开展？
- 渠道销售是否能及时知道哪些网点需要加强投教陪伴和服务支持？

因此，本课题希望建立一个能够持续追踪活动后渠道服务效果的分析工具。

## 四、核心分析视角

本课题重点选择：

**活动后 T+7、T+30、T+90 三阶段服务效果追踪**

分析不只关注活动当天表现，也不将 T+90 单独作为单场活动效果的精确归因依据，而是围绕活动后的不同阶段服务信号进行持续观察。

需要注意，T+90 距离活动时间较长，期间可能受到市场涨跌、产品净值波动、银行自身营销节奏、竞品活动、后续服务动作等因素影响。因此，T+90 更适合用于观察中期陪伴效果和活动模式沉淀，不适合单独证明某一场活动直接导致经营数据变化。

建议追踪时间点包括：

- T+7：活动反馈与服务承接，重点观察问卷反馈、渠道最需要的材料、风险提示是否充分、是否需要安排二次沟通。
- T+30：短期经营观察，重点观察保有量变化、赎回压力、渠道反馈集中问题、材料支持是否完成。
- T+90：中期复盘沉淀，重点观察保有量稳定性、网点客群盈利体验、活动主题是否值得复制、是否需要纳入持续陪伴名单。

## 五、指标体系

### 1. 活动投入层

用于描述活动本身的资源投入和活动类型。

可包含指标：

- 活动数量。
- 活动日期。
- 分行、支行或网点名称。
- 银行类型，例如分行、支行、网点，或国有行、股份行、城商行等。
- 活动规模，例如分行春秋大场 200-300 人，网点小场 10-20 人。
- 活动主题，例如市场观点、基金产品、指数投资、养老、信用卡、银行权益活动等。
- 讲师类型，例如基金公司讲师、银行讲师、投研人员、指数团队等。
- 渠道销售负责人或服务团队。

### 2. 渠道触达层

用于衡量活动触达的渠道范围和活动反馈。

可包含指标：

- 到场人数。
- 覆盖分行、支行或网点数量。
- 活动场次。
- 渠道反馈情况。
- 问卷反馈聚合结果。
- 后续投教服务需求。
- 后续材料支持需求。

### 3. 渠道经营观察层

用于观察活动后网点经营数据是否发生变化。该部分仅作为经营现象观察，不直接作为客户服务评价目标。

可包含指标：

- 活动后 30 天和 90 天网点保有量变化。
- 活动后 30 天和 90 天渠道申购、赎回的聚合变化。
- 保有量稳定性。
- 赎回压力变化。
- 同类网点或同类活动之间的聚合对比。
- 活动前后网点整体咨询、投诉或反馈数量变化，如数据可得。

### 4. 网点客群盈利体验层

用于判断网点客群整体持有体验是否改善，不追踪个人投资者。

可包含指标：

- 网点客群整体收益率区间。
- 盈利客户占比或亏损客户占比，如有聚合数据。
- 浮亏收窄情况，如有聚合数据。
- 赎回压力是否与市场波动或盈利体验相关。
- 渠道反馈中关于客户持有体验的问题。
- 网点是否需要补充市场观点、风险解释或产品陪伴材料。

### 5. 问卷反馈层

用于在 T+7 阶段快速收集渠道对活动质量和投教内容的反馈。考虑到活动后客户和渠道填写意愿有限，问卷应尽量短，以选择题为主。

建议采用：

- 6 道必答选择题。
- 1 道选填开放建议。
- 预计填写时间控制在 45-60 秒。
- 不采集个人身份信息。
- 结果只按活动 ID、分行、支行、网点等渠道聚合口径保留。

可量化指标包括：

- 活动质量评分。
- 投教观点质量评分。
- 主题匹配评分。
- 风险提示充分度。
- 后续材料需求。
- 后续服务方式偏好。
- 渠道反馈摘要。

## 六、Agent/Skill 设计思路

经过讨论，建议采用：

**对外讲 Agent，对内先实现 Skill。**

原因是当前任务流程比较稳定，适合先封装成一个可复用的业务分析 Skill，再由 Agent 调用。

底层 Skill 负责标准化分析流程：

1. 读取活动、渠道、网点经营、问卷反馈等聚合数据。
2. 按分行、支行、网点和活动类型匹配活动后经营数据。
3. 在 T+7、T+30、T+90 三个节点分别生成活动后服务信号。
4. 计算活动后 30 天和 90 天的保有量变化。
5. 判断网点客群盈利体验和服务反馈是否改善。
6. 识别需要加强陪伴的网点、表现较好的活动模板和可沉淀的投教内容。
7. 生成渠道销售服务提醒、飞书推送文案、邮件正文和 Excel 附件。

上层 Agent 负责理解渠道销售或管理层的问题，并调用 Skill 生成结果，例如：

- “帮我看这场活动 T+7/T+30/T+90 的服务信号。”
- “哪些网点需要加强投教陪伴？”
- “哪个网点的活动反馈和后续表现更好？”
- “下次活动主题应该怎么调整？”

## 七、Demo 最小可跑通方案

Demo 不需要一开始做完整系统，可以先基于几张模拟或整理后的表格跑通闭环。

### 输入数据

建议准备原始数据表，全部采用渠道聚合口径。实际工作中不假设能直接获得已经整理好的 `06_Skill输入汇总` 宽表，该宽表应由 Skill 自动生成，作为中间结果或输出附件。

1. 活动表
   - 活动 ID
   - 活动日期
   - 分行/支行/网点
   - 银行类型
   - 活动规模
   - 活动主题
   - 到场人数
   - 讲师类型
   - 渠道销售负责人或服务团队

2. 渠道经营数据表
   - 分行/支行/网点
   - 活动 ID
   - 活动前保有量
   - 活动后 30 天保有量
   - 活动后 90 天保有量
   - 活动后 30 天申购金额，聚合口径
   - 活动后 90 天申购金额，聚合口径
   - 活动后 30 天赎回金额，聚合口径
   - 活动后 90 天赎回金额，聚合口径
   - 网点客群整体盈利区间或盈利占比，如数据可得
   - 渠道咨询、投诉或反馈数量，如数据可得

3. 问卷模板表
   - 问题编号
   - 问题文本
   - 题型，例如单选、多选、选填
   - 选项
   - 是否必答
   - 量化方式
   - 进入反馈表字段

4. 问卷与反馈表，可选
   - 活动 ID
   - 分行/支行/网点
   - 活动质量评分
   - 投教观点质量评分
   - 主题匹配评分
   - 风险提示充分度
   - 渠道反馈
   - 终端客户反馈聚合结果，如数据可得
   - 后续投教需求
   - 后续材料支持需求

### 内部生成字段

Skill 读取上述原始表后，应自动生成一张分析宽表，包含：

- 活动基本信息。
- T+30 和 T+90 保有量变化。
- T+30 和 T+90 申购、赎回等聚合观察指标。
- 赎回压力。
- 网点客群盈利占比。
- 服务效果标签。
- 材料需求。
- 材料需求来源。
- 建议输出。

其中，材料需求来源应明确标注，避免在没有问卷的情况下凭空判断。

材料需求来源分为三类：

- 问卷反馈：渠道或活动参与方明确选择后续所需材料，置信度最高。
- 经营信号推断：没有问卷时，根据保有量变化、赎回压力、盈利占比等经营观察指标推断。
- 活动主题映射：问卷和经营信号都不足时，根据活动主题进行基础映射。

示例推断规则：

- T+30 赎回压力高或保有量短期下降：优先建议“波动市场客户问答”或“产品风险收益解释页”。
- T+90 盈利占比较低或保有量下降：优先建议“产品风险收益解释页”或持有体验陪伴材料。
- 指数投资主题：优先建议“指数投资讲稿”。
- 养老投教主题：优先建议“养老投教折页”。
- 市场展望或资产配置主题：优先建议“月度市场观点材料”。

### 数据缺失时的降级方案

如果部分数据拿不到，Skill/Agent 应按数据可得性降级分析：

- 如果有网点级经营数据：做网点活动后 30/90 天服务效果追踪。
- 如果只有支行或分行级数据：做支行/分行聚合对比，不下钻到网点。
- 如果没有申购、赎回明细：只分析保有量变化、问卷反馈和渠道服务需求。
- 如果没有盈利体验数据：用渠道反馈、赎回压力和投教需求作为替代观察指标。
- 如果只有活动与问卷数据：做活动内容复盘和后续服务建议，不做经营效果归因。
- 如果没有问卷数据：用经营信号推断和活动主题映射生成材料需求，并在飞书提醒和 Excel 附件中标注来源。

### 输出结果

Skill/Agent 应输出：

- T+7、T+30、T+90 三阶段服务效果摘要。
- 30 天和 90 天保有量变化分析。
- 网点客群盈利体验或替代指标分析。
- 需要加强陪伴的网点清单。
- 优秀案例。
- 渠道销售服务提醒。
- 飞书机器人提醒文案。
- 定期邮件正文。
- Excel 附件，包含节点提醒明细、活动汇总分析、重点关注、优秀案例、材料需求统计、图表数据和自动生成的分析宽表。
- 后续投教内容和材料支持建议。
- 可用于 PPT 的活动复盘摘要。

## 八、自动服务提醒设计

自动提示渠道销售是本课题的亮点之一。系统不只是生成静态报表，而是根据渠道聚合数据变化主动给出服务建议。

### 1. 提醒触发机制

建议采用 **T+7、T+30、T+90 三节点提醒**，而不是只在 T+90 进行一次复盘。

三个节点都可以通过飞书机器人进行提醒，但提醒内容和强度不同：

- T+7：轻提醒，偏活动反馈与服务承接。
- T+30：重点提醒，偏短期经营观察和陪伴需求识别。
- T+90：复盘提醒，偏活动模式沉淀和长期服务策略。

优先级不决定是否提醒，也不决定是否进入周报。所有活动在对应节点都应生成提醒和记录。优先级只影响：

- 飞书消息颜色和排序位置。
- 建议完成时限。
- 是否需要负责人确认处理。
- 是否进入周报中的“本期重点关注”模块。

### 2. 飞书提醒设计

飞书机器人提醒可以作为渠道销售的及时触达方式。实际接入时，可通过飞书群机器人 webhook 定时或按触发条件发送；Demo 阶段可先生成飞书 Markdown 风格文案和 JSON payload，用于模拟上线效果。

提醒内容应尽量短，适合飞书阅读。建议每条控制在 5 行左右，包含：

- 渠道名称和活动信息。
- 当前节点，例如 T+7、T+30、T+90。
- 观察信号，例如问卷评分、材料需求、保有量变化、盈利占比、赎回压力等。
- 保有量变化数据，例如 T+30、T+90 保有量变化率和赎回压力。
- 建议服务动作，例如补充材料、安排线上短会、沉淀优秀案例、优化讲师材料。
- 材料需求来源，例如问卷反馈、经营信号推断、活动主题映射。
- 建议完成时限。

示例提醒包括：

- T+7：某网点活动反馈已生成，渠道最关注“波动市场客户问答”，建议本周补充相关材料。
- T+30：某支行活动后保有量波动较大，渠道反馈集中在风险收益解释，建议安排一次线上短会。
- T+90：某网点小场活动复盘完成，保有量稳定、反馈较好，建议沉淀为优秀案例。

### 3. 邮件与附件设计

实际工作中建议采用“邮件正文 + Excel 附件”的形式，而不是把所有内容写成长篇周报正文。

邮件正文只保留：

- 一句话结论。
- 本期概览。
- 重点关注。
- 优秀案例。
- 下周建议动作。

完整数据放入 Excel 附件，附件建议包含：

- 邮件摘要。
- 节点提醒明细。
- 活动汇总分析。
- 重点关注 Top 5。
- 优秀案例 Top 3。
- 材料需求统计。
- 图表数据。
- 自动生成的分析宽表。

所有活动结果都需要进入邮件附件或阶段复盘，而不是只收录高优先级提醒。

原因包括：

- 稳定提升型活动可沉淀为优秀案例。
- 观察跟踪型活动有助于判断哪些主题和渠道需要继续观察。
- 需加强陪伴型活动需要团队复盘服务动作是否及时。
- 内容/陪伴优化型活动有助于优化投教材料、讲师安排和活动主题。

邮件附件或阶段复盘建议包括：

- 本期活动数量。
- 分行、支行、网点活动分布。
- 大场、中型场、小场活动分布。
- T+7/T+30/T+90 提醒数量。
- 稳定提升活动案例。
- 需加强陪伴网点清单。
- 高频后续材料需求。
- 下阶段渠道服务建议。

### 4. 重点关注与优秀案例选择逻辑

重点关注 Top 5 不应简单按原始顺序截取，而应按“关注分”排序。关注分可综合考虑：

- 优先级，高优先级权重最高。
- 节点，T+30 短期经营观察和 T+90 中期复盘权重略高。
- 渠道层级，分行和支行影响范围更大。
- T+30 保有量是否下降。
- T+90 保有量是否下降。
- 赎回压力是否偏高。
- T+90 盈利占比是否偏低。
- 活动质量或投教观点评分是否偏低，如有问卷。

为提升邮件可读性，重点关注 Top 5 建议每场活动只保留一个最高关注分节点，避免同一活动在邮件正文中重复出现。

优秀案例 Top 3 应按“优秀案例分”排序。优秀案例分可综合考虑：

- T+90 保有量变化较好。
- T+90 盈利占比较高。
- 赎回压力较低。
- 活动质量和投教观点评分较高，如有问卷。
- 小场活动优先，因为更适合沉淀为可复制的网点陪伴模板。
- 服务效果标签为稳定提升。

示例提醒包括：

- 某网点活动后 90 天保有量较为稳定，且投教观点评分较高，可考虑延续同类主题活动。
- 某网点活动后赎回压力上升，建议补充市场波动解释和产品风险收益特征材料。
- 某分行大场活动触达广，但 90 天保有量改善有限，建议增加活动后网点二次陪伴。
- 某类小场活动虽然人数较少，但保有量稳定性更好，可作为优秀案例沉淀。
- 某些渠道对投教观点评分高，建议沉淀为标准化讲稿或月度陪伴材料。

可将渠道对象分为几类：

- 稳定提升型网点：活动后保有量稳定或提升，反馈较好，可沉淀为优秀案例。
- 需加强陪伴型网点：保有量波动较大、赎回压力上升或渠道反馈集中，适合补充投教支持。
- 内容优化型网点：问卷显示活动质量或观点质量有提升空间，需优化主题和讲师匹配。
- 持续支持型分行：活动规模大、覆盖面广，适合建立常态化月度观点陪伴机制。

## 九、预期业务价值

该方案的业务价值包括：

- 帮助渠道销售从“活动结束”转向“活动后持续服务”。
- 帮助渠道判断哪些活动更有助于客户理解和长期陪伴。
- 帮助管理层评估不同银行、不同网点、不同活动形式的服务效果。
- 帮助渠道销售及时发现需要补充投教材料、市场观点或风险解释的网点。
- 帮助沉淀标准化投教内容和高质量活动模板。

## 十、后续 PPT 汇报思路

后续 PPT 可以按照以下逻辑展开：

1. 背景痛点：渠道活动多，但复盘偏短期、偏表层。
2. 合规边界：客户服务不与销量直接挂钩，数据分析采用渠道聚合口径。
3. 课题目标：追踪活动后 T+7、T+30、T+90 三阶段渠道服务效果和网点客群盈利体验。
4. 解决方案：构建渠道活动后网点服务效果追踪 Agent。
5. 分析框架：活动投入、渠道触达、经营观察、盈利体验、问卷反馈、服务提醒。
6. 技术实现：底层 Skill 封装分析流程，上层 Agent 理解问题并调用 Skill。
7. Demo 展示：输入原始渠道聚合表格，输出飞书提醒、邮件正文、Excel 附件和复盘结论。

## 十一、暂定一句话总结

本课题希望构建一个“渠道活动后网点服务效果追踪 Agent”，在客户服务不与销量直接挂钩、不使用个人客户 ID 的合规前提下，通过追踪渠道活动后 T+7、T+30、T+90 三个节点的活动反馈、服务承接、网点保有量变化、网点客群盈利体验和渠道反馈，自动生成飞书服务提醒、邮件正文、Excel 附件和优秀案例建议，帮助渠道活动从一次性触达升级为持续化渠道服务闭环。
